# Multiview Hallucination Detection via Nadler Spectral Fusion
## Experiments Report

*Omri Segev | Thesis supervised by Bracha Laufer-Goldshtein & Ofir Lindenbaum*

---

## Background

This thesis investigates whether **Nadler spectral fusion** can improve hallucination detection in LLMs by aggregating multiple uncertainty signals ("views") of the same question. The appeal is that Nadler is **unsupervised**: it estimates each view's reliability from the covariance structure of the signals alone, without ground-truth labels at inference time.

**The core hypothesis**: if you compute uncertainty scores for the same question from K different perspectives, and fuse them with Nadler, the ensemble should outperform any single view — because different perspectives expose different failure modes.

Two uncertainty measures are explored:
- **Semantic Entropy (SE)** — samples multiple stochastic answers, clusters by meaning via NLI, measures entropy over the cluster distribution (Farquhar et al., Nature 2024)
- **EPR (Entropy Production Rate)** — mean token-level entropy of a single greedy generation (artefactual library, K=15 log-probs)

**Nadler requires two conditions to produce positive lift:**
1. **Conditional independence** — views must make errors on different questions (low pairwise Spearman ρ)
2. **Common target** — all views must predict the correctness of the same model on the same question

Violating either condition produces negative lift. This framework drove the entire experimental design.

## Experiment 1 — Semantic Entropy with Diverse Views

**Model**: Llama-based | **Datasets**: TriviaQA, SQuAD | **Labels**: gold answers

**Views used**:
| View | Type |
|------|------|
| Original SE | Entropy (sampling-based) |
| Formal rephrasing | Linguistic variation |
| Simple rephrasing | Linguistic variation |
| French translation | Linguistic variation |
| German translation | Linguistic variation |
| Verify ("Are you sure?") | Behavioral — logical consistency |
| Skeptic ("Does this contain errors?") | Behavioral — logical consistency |

**Results**:
| Dataset | Baseline SE | Ensemble | Lift |
|---------|------------|----------|------|
| TriviaQA | 76.9% | 81.8% | **+4.9%** |
| SQuAD | 74.6% | 77.4% | **+2.8%** |

**Key finding**: Nadler fusion produced consistent positive lift. The Verify and Skeptic views are behaviorally different from the linguistic views — they test whether the model maintains its answer under challenge (*logical consistency*) rather than how uncertain the token probabilities are. This signal diversity is hypothesized as the key driver of the lift.

## Experiment 2 — EPR with Prompt Template Views (v1 → v3)

**Models**: Ministral-8B, Falcon-3-10B, Phi-4 | **Datasets**: TriviaQA, WebQuestions (200 samples each) | **Labels**: LLM-as-Judge (Qwen2.5-7B) | **Temperature**: T=1.0

Three generations of view design were tested, all at T=1.0:

**V1 — Multilingual** (formal/simple/German/French, generated by the model itself):
Negative lift across all models and datasets. Language shift corrupts EPR because token probability distributions change fundamentally when the question is in a different language.

**V2 — English prompt templates** (formal, factual, one-word, expert, direct, confident):
Still negative lift. Removing translation fixed the language problem but not the underlying correlation issue.

**V3 — Optimized English templates** via exhaustive C(11,4)=330 subset search:
Best combination found: `['direct', 'stepwise', 'paraphrase', 'concise']`. Fusion AUC = 0.935 on 30 samples, but on the full 200-sample evaluation: **still negative lift** across all 6 model×dataset combinations.

**Full results (v3, T=1.0):**
| Dataset | Model | Single EPR | Nadler Lift |
|---------|-------|-----------|-------------|
| TriviaQA | Ministral-8B | 75.1% | −2.2% |
| TriviaQA | Falcon-3-10B | 80.8% | −5.7% |
| TriviaQA | Phi-4 | 73.4% | −3.1% |
| WebQ | Ministral-8B | 68.9% | −7.5% |
| WebQ | Falcon-3-10B | 68.1% | −1.4% |
| WebQ | Phi-4 | 62.9% | −1.1% |

**Diagnosis**: All prompt template variants trigger the same parametric knowledge in the same model — RLHF alignment compresses model responses into a narrow distribution regardless of instruction phrasing. Pairwise Spearman ρ between any two views: **>0.8**. Nadler's conditional independence condition is violated. No selection algorithm (greedy disagreement, DPP MAP, exhaustive search) can rescue a candidate pool where all views are fundamentally correlated.

## Experiment 3 — Multi-Model EPR Ensemble

**Setup**: Fuse EPR signals from Falcon-3-10B, Ministral-8B, and Phi-4 using Nadler. No new inference needed — loaded existing v3 checkpoints. Labels: majority vote (≥2/3 models correct).

**Pairwise Spearman ρ:**
| Pair | TriviaQA | WebQ |
|------|---------|------|
| Ministral ↔ Falcon | 0.355 | 0.338 |
| Ministral ↔ Phi-4 | 0.307 | 0.465 |
| Falcon ↔ Phi-4 | 0.432 | 0.258 |

Correlations are very low (0.26–0.47) — Nadler's independence condition IS satisfied. Yet lift is **still negative** across all combinations (−0.8% to −6.8%).

**Root cause**: Nadler's second condition is violated. Ministral's EPR predicts whether *Ministral* answered correctly. Falcon's EPR predicts whether *Falcon* answered correctly. These are different targets — fusing them and evaluating against a single model's labels means the other models' signals are noise.

**Unified diagnosis at this point:**
| Experiment | Cond. independence (ρ < 0.6) | Common target | Lift |
|------------|------------------------------|---------------|------|
| Prompt templates (v3) | ✗ ρ > 0.8 | ✓ | Negative |
| Multi-model ensemble | ✓ ρ ≈ 0.3 | ✗ | Negative |
| **What is needed** | ✓ | ✓ | ? |

**Implication**: the required approach is signals that are (a) decorrelated AND (b) all computed from the same single model on the same question. This pointed directly to temperature-varied EPR and behavioral probing.

## Experiment 4 — Temperature-Varied EPR

**Model**: Falcon-3-10B | **Datasets**: TriviaQA + WebQ (200 each) | **Labels**: gold string matching (no judge) | **Notebook**: `Temperature_EPR_Ensemble.ipynb`

**Rationale**: Different sampling temperatures are non-linear transformations of the same logit distribution, capturing different uncertainty components:
- **T=0.3** — mode certainty: how peaked is the distribution?
- **T=1.0** — paper default: standard generation entropy
- **T=1.5** — mode fragility: how quickly does the distribution spread under pressure?
- **T=2.0** — noise floor: how diffuse does the distribution become?

All views predict Falcon's correctness on the same questions → common target satisfied.

**Single-view AUC:**
| Temp | TriviaQA | WebQ |
|------|---------|------|
| T=0.3 | 71.6% | 64.4% |
| **T=1.0 (baseline)** | **79.1%** | 71.8% |
| T=1.5 | 74.9% | **73.0%** |
| T=2.0 | 72.5% | 66.3% |

**Pairwise Spearman ρ** (vs >0.8 for prompt templates):
- Range: **0.38–0.75** — significantly lower
- Most decorrelated: T=0.3 ↔ T=2.0 (ρ=0.43 / 0.38)

**Fusion results (Nadler):**
| Configuration | TriviaQA | Lift | WebQ | Lift |
|--------------|---------|------|------|------|
| All 4 temps | 80.7% | **+1.6% ✓** | 74.7% | **+2.9% ✓** |
| Best 3-view (WebQ): T=1.0+T=1.5+T=2.0 | — | — | 74.2% | +2.4% ✓ |
| Any 2-view pair | 42–57% | −28 to −34% | — | catastrophic |

**Key findings**:
1. **First consistent positive lift** — validates the theoretical framework
2. **2-view collapse is catastrophic** — Nadler with 2 views has a single off-diagonal covariance value; ambiguous binarization inverts the signal entirely. Rule established: Nadler requires ≥3 views
3. **More views = more lift** — all-4 is best on both datasets
4. T=1.5 outperforms T=1.0 as a standalone on WebQ (+1.2pp) — dataset-dependent optimal temperature

## Experiment 5 — Temperature + Behavioral Views (Verify / Skeptic)

**Model**: Falcon-3-10B | **Datasets**: TriviaQA + WebQ (200 each) | **Labels**: gold string matching | **Notebook**: `Temperature_EPR_Ensemble.ipynb` (18 cells after patch)

**Motivation**: The SE experiment's lift was partly driven by Verify and Skeptic views — behavioral signals measuring logical consistency rather than generation entropy. Can the same approach be applied to EPR in a gray-box / API-compatible way?

**Implementation**:
- **Verify**: `P(Yes | "Is this answer correct?")` — first-token log-prob, single forward pass
- **Skeptic**: `1 − P(Yes | "Does this answer contain errors?")` — inverted doubt signal
- No hidden states, no fine-tuning — fully API-compatible

**Single-view AUC (all 6 views):**
| View | TriviaQA | WebQ |
|------|---------|------|
| T=0.3 | 71.6% | 64.4% |
| T=1.0 (baseline) | 79.1% | 71.8% |
| T=1.5 | 74.9% | 73.0% |
| T=2.0 | 72.5% | 66.3% |
| **Verify** | **80.0%** | 69.7% |
| **Skeptic** | 76.3% | **74.5%** |

Verify is the strongest single view on TriviaQA (80.0%), beating T=1.0 EPR. Skeptic is the strongest single view on WebQ (74.5%).

**Spearman ρ — behavioral vs temperature views:**
| Pair | TriviaQA ρ | WebQ ρ |
|------|-----------|-------|
| Verify ↔ T=0.3 | 0.444 | **0.201** |
| Verify ↔ T=1.0 | 0.627 | 0.374 |
| Skeptic ↔ T=0.3 | **0.322** | **0.203** |
| Verify ↔ Skeptic | 0.783 | 0.666 |

Behavioral views are substantially decorrelated from temperature-varied EPR, especially on WebQ (ρ as low as 0.20).

**Fusion results:**
| Configuration | TriviaQA | Lift | WebQ | Lift |
|--------------|---------|------|------|------|
| All 4 temps (previous) | 80.7% | +1.6% | 74.7% | +2.9% |
| T=1.0 + Verify + Skeptic | 75.1% | −4.1% | 72.2% | +0.4% |
| **All 6 (4 temps + Verify + Skeptic)** | **81.5%** | **+2.4%** | **76.0%** | **+4.2%** |
| Best 3-view (WebQ): T=1.5 + Verify + Skeptic | — | — | 75.9% | **+4.1%** |

**Best result achieved**: WebQ 76.0% vs paper baseline 68.2% = **+7.8% absolute over the original EPR method**.

**Key findings**:
1. Behavioral views add lift consistently on top of temperature ensemble — additive effect
2. Behavioral views alone fail (`T=1.0 + Verify + Skeptic` = −4.1% on TriviaQA) — Verify and Skeptic are correlated with each other (ρ=0.66–0.78), they need a diverse temperature base
3. Efficient 3-view config: `T=1.5 + Verify + Skeptic` achieves 75.9% (+4.1%) on WebQ — near-matching all-6 at half the cost

## Experiment 6 — Prompt Template Views at T=1.5 (Ablation)

**Models**: Ministral-8B, Falcon-3-10B, Phi-4 | **Datasets**: TriviaQA + WebQ (200 each) | **Labels**: LLM-as-Judge (Qwen2.5-7B) | **Notebook**: `Multiview_EPR_T15.ipynb`

**Question**: Was the negative lift in Experiment 2 partly caused by the T=1.0 choice, or is it fundamental to prompt-template variation?

**Setup**: Identical to Experiment 2 (v3 views: direct, stepwise, paraphrase, concise), only `TEMPERATURE = 1.0 → 1.5`.

**Results:**

*TriviaQA:*
| Model | Paper EPR | Our EPR (T=1.5) | Nadler | Lift |
|-------|-----------|-----------------|--------|------|
| Ministral-8B | 81.4% | 80.2% | 83.2% | **+3.0%** |
| Falcon-3-10B | 75.4% | 83.2% | 83.7% | **+0.5%** |
| Phi-4 | 78.2% | 74.1% | 75.0% | **+1.0%** |

*WebQuestions:*
| Model | Paper EPR | Our EPR (T=1.5) | Nadler | Lift |
|-------|-----------|-----------------|--------|------|
| Ministral-8B | 65.4% | 72.2% | 66.5% | −5.7% |
| Falcon-3-10B | 68.2% | 74.7% | 70.8% | −3.9% |
| Phi-4 | 65.2% | 73.2% | 67.6% | −5.6% |

**Key findings**:
1. **T=1.5 reverses the lift sign on TriviaQA for all three models** — from negative at T=1.0 to consistently positive at T=1.5. Temperature was a contributing factor to the null result in Experiment 2.
2. **WebQ remains negative and worsens** (−3.9 to −5.7%). Short WebQ questions generate fewer tokens; at T=1.5 those tokens are noisier, degrading EPR signal quality faster than it decorrelates.
3. **Individual AUCs rise significantly at T=1.5** — Falcon-3-10B reaches 83.2% on TriviaQA vs paper's 75.4%, suggesting T=1.5 genuinely improves EPR discriminability on longer factoid questions.

## Conclusions

### 1. Two conditions must hold simultaneously for Nadler to work

Experiments 2 and 3 provided a clean diagnostic:
- Prompt templates at T=1.0 satisfied the common-target condition but violated independence (ρ>0.8) → negative lift
- Multi-model ensemble satisfied independence (ρ≈0.3) but violated common target → negative lift
- Temperature variation and behavioral views satisfy **both** → positive lift

This is the central empirical contribution: a precise, two-condition diagnostic for when Nadler spectral fusion succeeds or fails.

### 2. Prompt-template variation is fundamentally limited at T=1.0

All prompt template variants share the same latent belief state — RLHF alignment means the model produces highly correlated EPR scores regardless of instruction phrasing. No selection algorithm (greedy disagreement, DPP MAP, exhaustive search over 330 subsets) changed this. The ceiling is hard.

**However**, at T=1.5 the inter-view correlation drops enough to produce positive lift on TriviaQA (+0.5 to +3.0%). Temperature choice is a meaningful lever for the prompt-template approach.

### 3. Temperature variation is a valid and practical diversity mechanism

Sampling the same question at T=0.3, 1.0, 1.5, 2.0 produces genuinely decorrelated EPR signals (ρ=0.38–0.75) while all views share the same ground truth label. This is the minimum setup that satisfies both Nadler conditions.

Key properties:
- No prompt engineering required
- No additional model loading — same weights, different generation temperature
- Lift is consistent across datasets (+1.6% TriviaQA, +2.9% WebQ)
- T=1.0 is best standalone on TriviaQA; T=1.5 is best on WebQ — no universal optimum

### 4. Behavioral views (Verify / Skeptic) add genuine orthogonal signal

First-token `P(Yes)` from reflective prompts ("Is this answer correct?") measures *semantic self-consistency* — a fundamentally different computational pathway from generation entropy. Key evidence:
- Verify achieves 80.0% standalone AUC on TriviaQA — matching T=1.0 EPR as a standalone predictor
- Spearman ρ with temperature views: 0.20–0.63 (much lower than any prompt template pair)
- Adding them to the temperature ensemble adds +0.8% on TriviaQA and +1.3% on WebQ
- Best single configuration: all-6 (4 temps + Verify + Skeptic) reaches 76.0% on WebQ vs paper's 68.2% — **+7.8% absolute**

Critically, these signals are fully gray-box / API-compatible (single forward pass, first-token log-probs only).

### 5. WebQ is consistently harder than TriviaQA

WebQ questions are short and open-ended (typically 5–10 words). At elevated temperatures, fewer generated tokens means higher-variance EPR estimates — the signal degrades before decorrelation benefits kick in. Prompt-template approaches (all temperatures) fail on WebQ, while the temperature+behavioral ensemble still works (+4.2%). This asymmetry is consistent across all experiments and models.

### 6. Nadler requires at least 3 views

Any 2-view Nadler fusion collapsed catastrophically (AUC near random, −28 to −34%). With 2 views the 2×2 off-diagonal covariance has a single value; the binarization step is ambiguous and frequently inverts the signal. This is a hard constraint on the minimum ensemble size.

### 7. Selection algorithm is not the bottleneck

DPP MAP selection (with a 1−1/e submodular guarantee) tied with naive disagreement-based selection. The ceiling on fusion performance is determined by the correlation structure of the candidate pool, not by which views are selected from it. A better selection algorithm cannot compensate for a pool of correlated views.

## Open Directions

1. **Hidden state probes as views** — layer-wise hidden states are architecturally decorrelated by construction (ρ near zero between early and late layers). Zero extra inference cost via forward hook. Connects to Ofir's VSDE work.

2. **T=2.0 prompt-template ablation** — completing the 3-point ablation (T=1.0 / T=1.5 / T=2.0) for the prompt-template framework. Would confirm whether higher temperature further reduces ρ between prompt views and improve lift on TriviaQA.

3. **Larger sample size** — current 95% CIs span 6–8pp; +1.6 and +2.9% lift is real but not conclusively significant. 500–1000 samples would tighten the bounds.

4. **Conformal calibration** — Bracha's LTT framework could wrap the fusion score with distribution-free guarantees on false-negative rate at inference time.

5. **Chain-of-thought EPR** — compute EPR on the reasoning trace separately from the final answer, as a structurally distinct signal.
