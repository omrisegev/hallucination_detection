Read this first0. What we measure and what we fuse

For each generated answer, we record the per-token entropy of the model's output distribution during a single forward pass — one number per generated token, no re-sampling. This gives us the entropy trace H(1), H(2), …, H(T).

From H(n) we extract 16 scalar features that characterise the trace's shape, frequency content, and dynamics. These 16 scalars are the inputs to the fusion algorithm; no ground-truth labels are used at inference time.

Stage A  ·  Divergence 1 — encoding (affects ALL subsequent steps)
How features are prepared before fusion

Idea (pseudo-code)

# Binary path
for each feature f:
    X_f[sample] = +1  if feat_f[sample] >= median(feat_f)
                = −1  otherwise
    # no explicit sign orientation — handled later by Assumption (iii)

# Continuous path
for each feature f:
    X_f[sample] = zscore( feat_f[sample] × sign_f )
    # sign_f = +1 if higher value → more likely correct
    #        = −1 if higher value → more likely wrong (e.g. high entropy = hallucination)

Implementation (Python)

# Binary path — sml_unsupervised() :588 → binarize_classifiers() :81
bin_dict = binarize_classifiers(feats_dict, feat_names)
# binarize_classifiers internals (per feature):
#   np.where(arr >= np.median(arr), 1, -1)
# No sign orientation — Assumption (iii) inside sml_fuse_signed handles it.
views = [bin_dict[f] for f in feat_names]   # k arrays of ±1


# Continuous path — lsml_continuous_pipeline() :580–584   ← ours
views = []
for f in feat_names:
    arr = np.array(feats_dict[f], dtype=float)
    s = FEATURE_SIGNS.get(f, +1)        # +1 or -1 from domain knowledge
    views.append(zscore(arr * s))       # orient then z-score
# k arrays of z-scored floats, one per feature

This encoding choice propagates through every subsequent step — R, S, group detection, and within-group SML are all computed from different numerical inputs. The covariance structure of ±1 inputs is different from that of z-scored continuous inputs, so the groups detected can differ too.

↓  encoded feature arrays X (±1 or z-scored)
Stage B  ·  detect_dependent_groups() :375 — step ①
Feature covariance R, then score matrix S · Jaffé–Fetaya–Nadler 2016, Eq. 15  ·  _score_matrix_lsml() :275

Idea (pseudo-code)

# Step ①: covariance matrix
R[i, j] = covariance(feature_i, feature_j)    # k × k

# Step ②: score matrix  (Eq. 15)
for each pair (i, j):
    S[i, j] = Σ_{k≠i,j} Σ_{l≠i,j,k} |R[i,j]·R[k,l] − R[i,l]·R[k,j]|
    # large S[i,j]: i and j co-vary in a way that can't be explained
    #   by their individual correlations with the rest of the features
    #   → evidence they share a latent group

Implementation (Python)

# detect_dependent_groups() :398–403
X = np.column_stack(views)    # n_samples × k
R = np.cov(X.T)               # k × k  (different for binary vs continuous — see Stage A)
#   binary:     Pearson correlations of ±1 arrays, bounded in [−1, 1]
#   continuous: covariance of z-scored floats, richer magnitude structure

# _score_matrix_lsml() :275 — how "linked" are features i and j relative to the rest?
s = np.zeros((m, m))
for i in range(m):
    for j in range(i + 1, m):
        total = 0.0
        for k in range(m):
            if k == i or k == j: continue
            for l in range(m):
                if l == i or l == j or l == k: continue
                total += abs(R[i,j] * R[k,l] - R[i,l] * R[k,j])
        s[i,j] = s[j,i] = total   # symmetric; high s[i,j] → same latent group

High S[i,j] means i and j co-vary in a way that cannot be explained by their individual correlations with the rest — they form a latent group. The double sum is O(k⁴); tractable for k = 5–16.

↓
Stage B  ·  detect_dependent_groups() :375 — steps ②③
Select K* groups · Eq. 14 residual  ·  _residual_lsml() :343

Idea (pseudo-code)

for K = 2, 3, … max_K:
    groups   = spectral_cluster(S, K)
    residual = how well the K-group block structure reproduces R    (Eq. 14)
               = Σ_{same group} (v_on[i]·v_on[j]  − R[i,j])²
               + Σ_{diff group} (v_off[i]·v_off[j] − R[i,j])²
    keep K with lowest residual → K*

# K*=1 (all features independent) is flat SML
# K*>1 means the hierarchical grouping explains R better than independence

Implementation (Python)

# detect_dependent_groups() :422–432
best_K, best_c, best_resid = None, None, float('inf')
for K in K_range:                               # K = 2, 3, … min(m, 8)
    c = _spectral_cluster_precomputed(s, K)     # SpectralClustering on s
    r = _residual_lsml(R, c)                    # Eq. 14:
    if r < best_resid:
        best_resid, best_K, best_c = r, K, c

# _residual_lsml() :343 — how well does the K-group model explain R?
resid = 0.0
for i in range(m):
    for j in range(m):
        if i == j: continue
        if c[i] == c[j]:                        # same group → within-group fit
            resid += (v_on[i]  * v_on[j]  - R[i, j]) ** 2
        else:                                   # different groups → cross-group fit
            resid += (v_off[i] * v_off[j] - R[i, j]) ** 2
# K=1 (all in one cluster) is the flat-SML independence assumption.
# A drop from resid(1) → resid(K*) proves the grouped model fits R better.

On MATH-500 with 16 features the algorithm typically selects K∈{4…6} — the correlated band-power features cluster together (|ρ| up to 0.88), while near-independent features like pe_mean are isolated into their own singleton group.

↓  K* groups, assignment c
Stage C  ·  within each group g
Within-group SML — Lemma 1 · sml_fuse_signed() :241

Idea (pseudo-code)

for each group g with features {f_1, …, f_{k_g}}:
    R_g  = covariance matrix of the group's features    # k_g × k_g
    R_off = R_g with diagonal zeroed                    # removes self-correlation
    v_g  = leading eigenvector of R_off
    # Lemma 1 (for ±1 inputs): v_i ∝ (2 × balanced_accuracy_i − 1)
    # Assumption (iii): flip v_g if majority of components are negative
    #                   → majority direction now means "more likely correct"
    score_g = group_features @ v_g     # weighted combination → one score per sample

Implementation (Python)

# lsml_continuous() :532–540 — called per group g (idx = feature indices in group)
score, w = sml_fuse_signed(*[X[:, i] for i in idx])

# sml_fuse_signed() :259–272 — the actual implementation:
X_g = np.column_stack(classifiers)    # n_samples × k_g
R = np.cov(X_g.T)                     # k_g × k_g covariance
R_off = R - np.diag(np.diag(R))       # zero the diagonal
_, vecs = eigh(R_off)
v = vecs[:, -1]                        # leading eigenvector: L2-unit-norm, signed
# Lemma 1 (Parisi–Nadler–Kluger 2014): for binary ±1 inputs, v_i ∝ (2α_i − 1)
# Assumption (iii) — unsupervised orientation:
if np.sum(v > 0) < k / 2:             # if most weights are negative, flip all
    v = -v
return X_g @ v, v                     # signed weighted sum + weight vector
# v is used with its sign — NOT abs-normalised.
# After the flip most v_i > 0; any remaining v_i < 0 subtract from the score.

Key detail: sml_fuse_signed returns (X @ v, v) where v is the raw signed L2-unit-norm eigenvector — not |v| / |v|.sum(). Positive components contribute, negative components subtract. The Assumption (iii) flip ensures that the majority direction corresponds to "higher score = more likely correct."

↓  score_g per group — before the encoding choice
Stage E  ·  Divergence 2 — virtual classifier encoding (second difference)
Binary path vs Continuous path: how score_g becomes ξ_g

Idea (pseudo-code)

# Both paths have score_g: a continuous weighted sum of group g's features.
# They differ only in what they pass to the next (cross-group) step:

# Binary path: collapse magnitude, preserve only sign
xi_g[sample] = +1  if score_g[sample] > 0
             = −1  otherwise   (Lemma 1 binary contract preserved)

# Continuous path: pass through as-is
xi_g[sample] = score_g[sample]   # keeps magnitude → +4.9 pp AUROC

Implementation (Python)

# Binary path — paper-faithful ————————————————  fusion_utils.py :475–476
xi_g = np.sign(score_g)
xi_g[xi_g == 0] = 1.0         # tie-break exact zeros to +1

# collapses virtual classifier to ±1 before cross-group step
# → Lemma 1 binary contract preserved  |  magnitude information discarded


# Continuous path — our adaptation ——————————  fusion_utils.py :539
xi_g = score_g                 # keep continuous weighted sum as-is

# → Lemma 1 binary contract not formally satisfied
#   R_off rank-1 fraction still 81–91%  (LSML_Theorem_Validation.html §1)
# → +4.9 pp macro AUROC over the binary path  (wins 25 of 29 cells)  ← ours

Everything above and below this point — group detection, sml_fuse_signed within groups, and sml_fuse_signed across groups — is shared code. Only this single assignment differs.

↓  K* virtual classifiers  ξ₁ … ξ_K*
Stage D  ·  cross-group
Cross-group fusion · lsml_continuous() :552  ·  calls sml_fuse_signed() :241

Idea (pseudo-code)

# ξ_1 … ξ_K* are (approximately) independent — each represents a different group.
# Apply the same SML logic one more time:
R_cross    = covariance(ξ_1, …, ξ_K*)         # K* × K*
R_off_cross = R_cross with diagonal zeroed
v_cross    = leading eigenvector of R_off_cross
# orient: Assumption (iii) flip (flip if majority negative)
fused = [ξ_1, …, ξ_K*] @ v_cross              # final prediction score per sample

Implementation (Python)

# The K* virtual classifiers are (approximately) independent of each other —
# each represents a different latent group.  Apply sml_fuse_signed once more:

fused, v_cross = sml_fuse_signed(xi_1, xi_2, …, xi_K)
# Same mechanics as within-group:
#   v_cross = leading signed eigenvector of cov([ξ₁,…,ξ_K]) off-diagonal
#   same Assumption (iii) orientation flip applies
# fused = virtual_arr @ v_cross   ← the returned score

The cross-group weights v_cross are signed — features (now virtual classifiers) that contribute negatively get a negative weight. On MATH-500 / Qwen-7B with 16 features: three strong groups get weights ≈ 0.30–0.34 each; the singleton pe_mean group gets ≈ 0.02, effectively switched off. The meta_dict returned by lsml_continuous() stores these in cross_weights — the source of the cluster-AUROC tables in Spectral_LSML_Report.html §7.

Formal status of the continuous adaptation. Two geometric properties the algorithm relies on hold for continuous z-scored inputs: the R_off rank-1 fraction is 81–91% across all 4 tested cells (T1 PASS — weight-estimation basis survives) and the Eq. 14 residual decreases monotonically with K in all 4 cells (T3 PASS — the grouping genuinely improves fit). The cost of keeping magnitude: virtual classifiers remain correlated across groups (T6 fails 3/4 cells). The strict Lemma 4 gap condition fails in finite samples for both binary and continuous inputs equally — a shared finite-sample limitation. Full test results: LSML_Theorem_Validation.html.  Net gain of the continuous path: +4.9 pp macro AUROC, winning 25 of 29 cells over the paper-faithful binary version.
