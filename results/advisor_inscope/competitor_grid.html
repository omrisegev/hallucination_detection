What the three method columns mean

A competitor's AUROC is only comparable to ours if it is paying the same price. These three columns say what each method actually needs, all read off the papers themselves:

  • Supervision — does it fit anything on hallucination labels? Ours never sees a label.
  • Access — the taxonomy is the LOS-Net paper's own: black-box generated text only; grey-box output distributions / log-probabilities; white-box model internals (hidden states, attention maps, activations), which the same paper calls “incomparable” to output-only methods. Ours is grey-box — it needs the top-k log-probabilities and nothing deeper.
  • Passes — generations per question at detection time. Ours is 1. A method at K=10 costs an order of magnitude more per question, which matters as much as AUROC for deployment.

Rows marked inferred are the ones whose paper is not in the local library, so the profile comes from the method's description elsewhere rather than from its own text.

How to read this

  • Selector@30 / Selector@16 are the trace-based Gated-Laplacian (a2.dufs) on the 30-view and 16-view pools; GroupFS@30 is the group-granular rule (a2.select) on the same gates.
  • All AUROC values are percentages, direction-resolved label-free against the epr anchor (see anchor robustness).
  • Every competitor number is re-derived from competitors_verified.csv; see provenance for which are verified.
  • The selector loses to the fixed subsets on macro. That is explained on the selection-headroom page: per-cell label-free selection is dominated by selection noise, while a subset fixed once across the whole corpus is a better-regularised estimator.
