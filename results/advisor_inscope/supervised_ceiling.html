Audit of how this number is computed

Three properties of the oracle code were checked before publishing it, because each could have inflated the ceiling:

CheckFinding
Per-fold AUROC is floored at 0.5 and direction-free (max(p, 1−p)), so an anti-predictive fold is flipped upward immaterial here — only 1 of 500 folds flip. Macro inflation is +0.07 pp on the 9-view set and +0.00 pp on the others, so the headroom framing stands.
Grouped folds actually applied on the multi-candidate cells confirmed for every k>1 cell.
The saturation filter silently drops views no GOOD_6 member dropped on any cell.
Regularisation strength C is fixed at 1.0 and never tuned a real caveat — across C ∈ {0.1, 1, 10} at 30 views the AUROC spread averages 1.24 pp and reaches 6.16 pp on the worst cell. The ceiling is therefore an untuned point estimate, not a tuned maximum.
