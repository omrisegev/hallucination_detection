Nothing on this site was published before this audit passed. It is here because two defects were found, both of which would have put wrong numbers in front of you.

1. Bench rows computed against a superseded feature cache

Two independent artifacts held per-cell AUROCs for the same subsets and disagreed. The disagreement was not in the code: driving both fusion paths on the same cell reproduces the same score, the same group count and the same residual to full precision. The cause was staleness. The benchmark harness is resume-safe — it skips any (variant, cell) pair already present in its output — so when a cell's feature cache was rebuilt, none of its existing rows was ever recomputed.

Sizing it required re-scoring every stored row's own selected subset against the current cache: 3821 rows re-scored, 258 stale, confined to 25 cells but spread across 6 of the benchmark files. A second, wider instance of the same defect was then found: on 11 further cells every learned selector had searched a pool four views smaller than the current one, so those selectors were handicapped. Both were fixed by deleting the affected rows and re-running.

2. Competitor numbers without citations

13 of the 19 published anchors carried no citation of any kind — no paper, table, page or scale. Every anchor that could be checked was checked against the raw extracted paper text, which surfaced two errors in how a competitor had been recorded. Full detail on the provenance page.

What is still open

  • The primary paper for the Gated-Laplacian selector is not in the local library, so that implementation is grounded only in a successor paper's description of it.
  • One competitor method still has no local PDF (Internal-States+RC); the LapEigvals / INSIDE / LOS-Net papers were all located and verified (61/62 rows, 17/18 anchors) — the LapEigvals PDF had been filed under its title, not the method name.
  • The supervised ceiling uses an untuned regularisation strength; the spread across plausible values reaches 6.2 pp on the worst cell.
