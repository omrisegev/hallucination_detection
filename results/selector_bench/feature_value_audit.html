Step 186-187 · label-free feature-subset selection

Feature value audit — 46-feature pool

Single-feature AUROC of every canonical feature per cell, per dataset, and overall — asking which features are noise that could leave the global pool. Answer: none are noise, but 13 carry the wrong fixed sign.

Question this page answers: which of the canonical-pool features carry signal anywhere, and which are noise everywhere and only dilute the pool. Scores are single-feature AUROCs under the pipeline's fixed offline orientation (the exact columns the fusion consumes) — a value below 50 means the fixed sign convention is wrong on that cell, not that the feature is worthless there. Verdicts are computed on the 46-view grid (19 cells); dropping a feature is a pool-curation decision that uses labels offline — that is allowed (it is design-time, like choosing the pool was), but it is not applied anywhere automatically: this page is analysis.

Verdict summary — the headline is orientation, not noise

The expected finding was “some features are random noise everywhere — drop them.” That is not what the data says: zero features are flat noise on this grid. The actual pathology: 13 of 30 features are consistently ANTI-oriented — strongly informative, but carried with the wrong fixed sign (mean AUC 0.27–0.45, i.e. genuinely 0.55–0.73 once flipped). The whole spilled-energy family and several spectral-shape features are in this bucket, and so is GOOD_5's own hl_ratio (0.378 here vs positive on the reasoning domains — see the cross-domain table below). L-SML absorbs a wrong sign through negative fusion weights, so bench results are unaffected; anything sign-sensitive (U-PCR's keep-mask, anchor-correlation priors, human reading of single features) is not.

What follows from this

  • Nothing needs deleting for being noise. The “drop the random features” hypothesis is answered: no feature is flat-noise across the grid, so pruning the pool buys nothing on signal grounds. The pool's real cost is selector burden, which the bench already prices in.
  • The sign convention deserves an offline audit. The 13 anti-oriented features (whole spilled-energy family included) are a candidate one-line offline fix in the sign tables — harmless to L-SML (weights absorb it), material to U-PCR and to any anchor-correlation objective (A4's anchor selector used |ρ|, so it was immune). Any change must be re-validated against the canonical repgrid scores before adoption.
  • Sign-unstable and domain-flipping features are the per-cell selection opportunity — concrete evidence that one fixed global subset (and one fixed sign) leaves per-cell value on the table, which is the thesis of the whole selection experiment.
  • Do not delete WEAK features: fusion exists to extract value from weakly-informative, decorrelated views.
