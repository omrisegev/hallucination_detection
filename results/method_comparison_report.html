Spectral Hallucination Detection — Method Comparison

29 evaluation cells across 5 domains · 8 pipeline variants · Run 2026-06-09

1. Glossary of Terms

Every variant name used in this report is defined here. Three design dimensions vary across variants: feature set, value encoding, and aggregation method.

Feature sets

NameFeaturesDescription
H1616 featuresAll 16 spectral entropy features computed from the token entropy sequence H(n): EPR, trace length, spectral entropy, band powers, dominant frequency, spectral centroid, STFT features, RPDI, sliding-window variance peak, permutation entropy, Hurst exponent, and two CUSUM statistics.
GOOD_55 featuresA subset selected in prior experiments: epr, low_band_power, sw_var_peak, cusum_max, spectral_entropy.

Value encoding

NameDescription
Binary (median threshold)Each feature value is thresholded at its median across the evaluation cell, producing a ±1 value per sample. Features where >40% of values equal the median ("saturated") are dropped before thresholding.
Binary + SignsSame as above, but sign orientation is applied first: features where higher value means less likely correct are multiplied by −1 before thresholding. Sign directions were derived from prior experiments.
Continuous + SignsFeatures are sign-oriented (as above) then z-score normalized (zero mean, unit variance). Raw normalized floats are passed to the aggregator — no binarization.

Aggregation methods

NameDescription
Flat SMLSpectral Meta-Learner: finds a single linear combination of all input classifiers via the leading eigenvector of their correlation matrix. No grouping step.
L-SML (Latent SML)First groups correlated classifiers into K clusters (K chosen by cross-validation), fuses within each group via eigenvector, then fuses the group scores via a second eigenvector. Proposed by Jaffé, Fetaya & Nadler (2016).
Simple AverageEqual-weight arithmetic mean of all sign-oriented feature values.
Oracle Best-1For each evaluation cell independently, picks the single feature from GOOD_5 with the highest AUROC. Not deployable (requires ground-truth labels to select the feature); included as an upper-bound reference.

7. Observations

  • Continuous encoding consistently outperforms binary encoding. CONT (70.1% macro) exceeds PROD (65.2% macro) by 4.9 pp. This holds in every domain: +4.6 pp (MATH-500), +4.9 pp (GSM8K), +0.7 pp (GPQA), +2.2 pp (RAG), +12.1 pp (QA). At the per-cell level, CONT exceeds PROD in 24 of 28 comparable cells.
  • In binary mode, Flat SML outperforms L-SML grouped fusion on the same 5-feature input. flat5 (68.3%) exceeds PROD/lsml_5_binary (65.2%) by 3.1 pp. In continuous mode the difference reverses but is small: CONT (70.1%) exceeds avg5 (69.7%) by 0.4 pp.
  • Reducing to 5 selected features improves performance versus using all 16. The improvement is +1.4 pp for L-SML-nosigns (65.4% → 66.8%) and +7.3 pp for Flat SML (61.0% → 68.3%).
  • Sign orientation does not consistently help in binary mode. lsml_5_nosigns (66.8%) outperforms lsml_5_signs_binary/PROD (65.2%) by 1.6 pp, despite PROD having explicit sign knowledge.
  • Simple average (69.7%) is nearly as strong as L-SML continuous (70.1%) in macro-average terms. The two differ by 0.4 pp. Domain-level results: avg5 leads on GPQA (57.5% vs 53.9%) and RAG (62.6% vs 61.7%); CONT leads on MATH-500 (84.3% vs 83.1%), GSM8K (75.6% vs 72.1%), and QA (75.0% vs 73.0%).
  • The Oracle Best-1 upper bound (71.8%) is 1.7 pp above the best deployable method (CONT, 70.1%). This means that, if an oracle selected the single best GOOD_5 feature per cell, the gain over CONT would be small.
  • GPQA is the hardest domain. All variants score between 50.2% and 58.9% on GPQA, near random chance. No variant clearly leads on GPQA: the best deployable method (CONT) ranks only 4th on that domain (53.9% vs avg5 57.5%).
  • RAG performance is consistently lower than math domains. All deployable variants score 56–63% macro on RAG versus 70–84% on MATH-500. GPQA similarly sits near 50–59%.
  • No GOOD_5 feature pair has mean Spearman |ρ| ≥ 0.75. The most correlated pair (epr | sw_var_peak) has mean |ρ| = 0.633 but exceeds the 0.75 threshold in 9 of 29 individual cells. All 10 pairs pass the L-SML rejection filter on average.
  • stft_max_high_power and stft_spectral_entropy saturate in most RAG cells. These two features produce uninformative binary splits on short answer traces (typical of retrieval-augmented QA). trace_length saturates on MATH-500 cells where the model frequently hits the max-token limit.
