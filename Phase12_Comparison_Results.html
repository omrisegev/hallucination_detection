What is L-SML v2?

L-SML stands for Latent Spectral Meta-Learner. We fuse spectral features of the token-entropy trace H(n) into a single hallucination score, fully unsupervised at test time.

v2 pipeline (Step 124, final):
  1. Pre-orient each feature with FEATURE_SIGNS — consensus signs derived once offline (Step 110) from cross-dataset majority vote. No test-time labels used.
  2. Binarize at empirical median via binarize_classifiers: above median → +1 ("likely correct"), below → −1.
  3. Filter to GOOD_FEATURES = ['epr', 'low_band_power', 'sw_var_peak', 'cusum_max', 'spectral_entropy'] — 5 features with consistently clean individual AUROC signal, selected offline via Step 121–123 ablation across 29 cells.
  4. Fuse with lsml_fuse (Algorithm 2, Jaffé-Fetaya-Nadler 2016): detect dependent classifier groups via spectral clustering, fuse within each group using SML, fuse across groups using SML.
Based on:
  • Parisi, Nadler, Kluger — PNAS 2014 (link): SML — leading eigenvector of off-diagonal covariance encodes classifier balanced accuracies without labels.
  • Jaffé, Fetaya, Nadler — 2016 (link): L-SML — extends SML to handle groups of correlated classifiers; no subset selection needed.
Feature selection rationale: The other 11 features have near-random or oscillating individual AUROC curves (signal is noise) and pollute the L-SML covariance matrix. Filtering to 5 clean features gives +1.7pp mean AUROC improvement overall. Binarization threshold (median vs optimized quantile) is a null result (+0.001pp); feature selection is what matters.

Labels are used only to compute the final AUROC — they never enter the fusion pipeline.
