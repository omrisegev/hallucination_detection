What is L-SML v2?

L-SML stands for Latent Spectral Meta-Learner. We fuse 16 spectral features of the token-entropy trace H(n) into a single hallucination score, fully unsupervised at test time.

v2 pipeline (Step 120):
  1. Pre-orient each feature with FEATURE_SIGNS — consensus signs derived once offline (Step 110) from cross-dataset majority vote. No test-time labels used.
  2. Binarize at empirical median via binarize_classifiers: above median → +1 ("likely correct"), below → −1.
  3. Fuse with lsml_fuse (Algorithm 2, Jaffé-Fetaya-Nadler 2016): detect dependent classifier groups via spectral clustering, fuse within each group using SML, fuse across groups using SML.
Based on:
  • Parisi, Nadler, Kluger — PNAS 2014 (link): SML — leading eigenvector of off-diagonal covariance encodes classifier balanced accuracies without labels.
  • Jaffé, Fetaya, Nadler — 2016 (link): L-SML — extends SML to handle groups of correlated classifiers; no subset selection needed.
Labels are used only to compute the final AUROC — they never enter the fusion pipeline.
